从本地新闻推荐到本地信息与服务

NewsBreak 正处于从「本地新闻推荐平台」演进为「本地信息与服务平台」的关键路径上。 这份分析回答两个问题：下一阶段缺什么系统，以及需要什么样的人来构建它。

01

战略背景

NewsBreak 正从「本地新闻推荐平台」演进为「本地信息与服务平台」，核心路径是：

Content → Intent → POI → Marketplace

当前系统已具备较强的本地内容推荐能力，但尚未形成完整的本地决策与服务系统。

02

系统演进阶段

Stage 1 已完成

Local Content Platform

内容推荐系统 · Content-centric system

核心能力

  • Feed 推荐系统
  • 本地新闻分发（geo-aware ranking）
  • 用户兴趣建模（content-based）
  • CTR / engagement optimization
Stage 2 当前缺失核心

Local Intent Platform

意图驱动的推荐系统 · Intent-centric system

目标能力

  • 用户本地意图识别（implicit intent）
  • Feed + Search 融合
  • content → action mapping
  • 本地决策排序系统
Stage 3 未来

Local Marketplace Platform

本地交易与履约系统 · Marketplace system

目标能力

  • POI / 商家系统
  • supply-demand matching
  • dispatch / routing / ETA
  • pricing & optimization
03

关键人才结构

围绕三阶段演进，对应三位核心负责人。

P0 · 立即招聘

Local AI / Intent System Owner

负责构建「用户在本地想做什么」的统一理解与排序系统。

核心职责

  • Feed + Search 融合架构
  • Local intent modeling
  • content → service / action 转化
  • ranking 从 CTR → decision optimization
  • real-time personalization

必备背景

  • 推荐系统 / 搜索排序经验（强）
  • Local / geo / LBS 相关经验（强加分）
  • 多目标优化 / embedding system

典型来源公司

字节跳动（推荐/搜索） Google（Search / Maps） 小红书（content→intent） Yelp（local search）
P0.5 · 同步启动

POI / Merchant Intelligence Owner

负责构建「现实世界有哪些商家/地点及其能力」的结构化理解系统。

核心职责

  • POI / entity system design
  • merchant graph / knowledge graph
  • geo relationship modeling
  • entity resolution & dedup
  • service taxonomy design
  • POI embedding system

必备背景

  • LBS / map / geo system
  • knowledge graph / entity system
  • large-scale structured data system

典型来源公司

美团（POI / 商家中台） Google（Maps / Knowledge Graph） 高德地图（地理系统） Yelp（local business graph）
P1 · 后置

Marketplace Optimization Owner

负责在供需约束下完成最优匹配与履约的系统设计。

核心职责

  • supply-demand matching
  • dispatch / routing / scheduling
  • ETA prediction
  • pricing / incentive system
  • real-time optimization / OR

必备背景

  • marketplace / logistics system
  • 运筹优化 / OR / RL
  • real-time system design

典型来源公司

Uber（dispatch / marketplace） DoorDash（delivery optimization） 滴滴出行（调度系统） Instacart（供需系统）
04

当前能力 vs 目标能力差距

✅ 已具备能力 · Stage 1

  • Feed recommendation system
  • Content understanding
  • Geo-aware news distribution
  • Engagement optimization

❌ 核心缺口

缺口 1 · 最大

Intent System 缺失

  • 无法理解用户「想做什么」
  • 仍停留在 content consumption
缺口 2 · 结构性

POI / World Model 缺失

  • 缺少商家/地点统一建模
  • 没有 knowledge graph
缺口 3 · 未来

Marketplace System 未建立

  • 无 supply-demand system
  • 无交易与履约体系
05

人才招聘优先级

  1. P0 · 立即招聘
    Local AI / Intent System Owner
    决定产品是否从「新闻平台」升级为「本地智能平台」。
  2. P0.5 · 同步启动
    POI / Merchant Intelligence Owner
    决定是否具备「现实世界理解能力」。
  3. P1 · 后置
    Marketplace Optimization Owner
    等交易系统成立后再引入。
06

关键结论

当前最大风险，不是推荐系统能力不足，而是：

缺少「Intent + POI」两层系统，
导致无法从内容平台升级为本地服务平台。
