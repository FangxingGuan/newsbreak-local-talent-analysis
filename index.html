从本地新闻分发到本地 AI 信息与服务平台

公司战略已升级：核心不再是「News」，而是「Local × AI」。 产品形态从单一新闻分发，演进为横向扩展的本地信息平台与纵向打通的本地服务平台。 这份分析回答：演进路径上需要什么样的算法与架构人才。

Consume 本地内容平台 已完成
→
Discover · Decide 本地 AI 信息平台 建设中 · 当前重点
→
Act 本地 AI 服务平台 同步启动
01

使命与愿景

Mission

让本地生活更易导航、决策与行动

每天为用户创造价值：在本地生活的方方面面，更轻松地 navigate（找）、decide（选）、act（做）。

Vision

成为本地生活的 AI 平台

连接人、信息与商业，构建更紧密、更知情、更有活力的本地社区。

02

六大战略锚点

理解公司方向的六个最清晰视角。

1

Local > News

Local 是核心，News 只是本地生活体验的其中一部分。

2

从新闻 App → AI/Tech 平台

构建以 Local × AI 为核心价值的应用与功能矩阵。

3

2C + 2B 双向赋能

除服务用户外，还要赋能 publisher、advertiser、本地商家等生态参与者；2B 同时验证并强化底层技术。

4

从 Information 到 Action

先「help people get informed」，再「help people get things done」。最终目标：用户任务完成数 + 本地商家可衡量业务结果。

5

自建 + 孵化投资

不仅自建，还通过投资部门支持构建本地内容、信息、商业生态的创新者。

6

难题优先于唾手可得

优先创造 Local × AI 长期核心价值。问题「难」往往是组织能力、人才、专业度的信号 — 要补齐。

03

发展路径：从本地新闻分发到本地 AI 平台

三阶段演进，对应用户价值的三种动作：消费 → 了解决策 → 行动。

阶段一 已完成

本地内容平台

Local News Distribution · Content-centric

用户价值：Consume — 看到本地的新闻与内容

已具备能力

  • Feed 推荐系统（geo-aware ranking）
  • 本地新闻分发与内容理解
  • 基于内容的用户兴趣建模
  • CTR / engagement 优化
阶段二 当前重点

本地 AI 信息平台

Local AI Information Platform · Intent-centric · 横向扩展

用户价值：Discover + Decide — 「help people get informed」

目标能力

  • 主应用升级：更多本地化 tab、支持孵化的架构、更强体验、更高内容与广告质量
  • 本地生活应用矩阵（content / info / services 三层）— breadth-first
  • 本地意图理解 · Feed + Search 融合
  • 主应用与垂类应用共享 API / 基建，互相反哺
  • 本地 AI 搜索与问答（信息侧）

结构性意义

同时具备防守（巩固本地 default entry point）与进攻（孵化新垂类、塑造用户行为）两面， 是当前最关键的战略之一。

阶段三 同步启动

本地 AI 服务平台

Local AI Commerce Platform · Action-centric · 纵向打通

用户价值：Act — 「help people get things done」

目标能力

  • 本地生活 AI agent 与搜索 — depth-first，一个垂类做透再扩展
  • Discover → Decide → Act 全链路闭环优化
  • 对接本地商家与服务提供方（BD + 生态投资）
  • POI / 商家系统、供需匹配、履约能力逐步建立
  • 成功指标：用户任务完成数 + 本地商家可衡量业务结果
04

两大产品平台 + 五大战略举措

两条主线（信息 / 服务）+ 一条共用底座（技术 / 生态），五大战略攻防互补。

平台 A · 横向

本地 AI 信息平台

横向扩展，覆盖本地内容、信息、服务。成功指标：有意义的新本地垂类被创造与验证的数量。

承载战略

  • 攻+守 战略 1：主应用 2.0 — 更多本地 tab、孵化架构、更强体验、内容与广告质量
  • 进攻 战略 2：本地生活应用矩阵（content / info / services） — breadth-first，hypergrowth + 可持续商业
平台 B · 纵向

本地 AI 服务平台

纵向打通用户与本地商家。成功指标：被连接 / 接入 / 活跃的本地商家与品类数量。

承载战略

  • 进攻 战略 3：本地 AI agent 与搜索 — depth-first，逐垂类优化 discover → decide → act 链路
底座 · 跨平台

技术底座与生态

支撑两大平台同时演进的技术与生态投入。

承载战略

  • 防守 战略 4：广告与基础技术体系 2.0 — A/B 测试基建、系统架构现代化、模型鲁棒性、广告算法与 2B 解决方案
  • 进攻 战略 5：本地生态投资 — 支持具有互补能力的初创公司
05

算法人才需求

三位核心算法负责人 — 对应「信息 / 服务 / 商业化」三条主线。

P0 · 立即招聘 · 算法

本地意图与搜推融合 Lead

负责构建「用户在本地想做什么」的统一理解与排序系统 — 信息平台的核心算法。

核心职责

  • Feed + Search 融合架构
  • Local intent modeling（implicit / explicit）
  • 排序从 CTR → decision optimization（导向决策与行动）
  • 主应用与垂类应用之间共享的用户表征 / embedding
  • 实时个性化与多目标优化

必备背景

  • 大规模推荐 / 搜索排序经验（强）
  • Local / geo / LBS 经验（强加分）
  • 多目标优化、embedding system、跨场景迁移

典型来源

头部短视频与内容社区（推荐/搜索） 大型搜索公司（Search / Maps） 本地点评 / 本地搜索平台
P0 · 立即招聘 · 算法

本地 AI Agent 与行动优化 Lead

负责把信息转化为任务完成 — 服务平台与 commerce 的核心算法。

核心职责

  • 本地生活 AI agent（discover → decide → act 闭环）
  • LLM + 工具使用 + 多步推理
  • 垂类纵深优化（一个垂类一个垂类做透）
  • 用户任务完成数 / 本地商家业务结果的算法侧 owner
  • RL / 反馈闭环 / 在线优化

必备背景

  • LLM / agent / tool use 系统经验（强）
  • 对话式或任务型 AI 落地经验
  • o2o / 本地服务业务理解（加分）

典型来源

大型基础模型团队 AI agent / copilot 产品团队 本地生活 / O2O 平台 AI 团队
P0 · 立即招聘 · 算法

广告算法与商业化科学 Lead

负责 ad 排序、定价与 2B 商业化算法 — 当前最确定的现金流，并支撑平台战略。

核心职责

  • 广告排序、出价、定价、归因
  • 面向 publisher / advertiser 的 2B 自助化算法
  • 跨应用矩阵的统一广告变现
  • 模型鲁棒性 / 反作弊 / 长期价值优化

必备背景

  • 大规模广告系统 / ad science 经验（强）
  • 多目标拍卖、calibration、uplift
  • 2B 平台 / 程序化广告经验（加分）

典型来源

大型广告平台 头部信息流广告团队 程序化广告 / DSP / SSP
06

架构人才需求

三位核心架构负责人 — 对应「应用矩阵 / 实验与 ML 平台 / 知识图谱 + Agent 运行时」。

P0 · 立即招聘 · 架构

应用矩阵与孵化平台架构师

负责主应用 + 多垂类应用的共享架构 — 让矩阵互相反哺成为可能。

核心职责

  • 跨应用共享 API、SDK、基建
  • 主应用「孵化器」模式的架构（新垂类 tab、模块化注入）
  • 多应用统一身份 / 数据 / 内容 / 广告通道
  • 新应用从 0 → 1 的发布流水线

必备背景

  • 多应用 / 多产品线平台架构经验
  • 移动端 + Web + 服务端全栈架构能力
  • API 与 SDK 设计、跨团队协作

典型来源

大型应用矩阵公司（多 App 共平台） 超级 App 平台架构团队 开发者平台 / 小程序平台
P0 · 立即招聘 · 架构

实验与 ML 平台架构师

负责 A/B 测试基建、特征平台、模型服务 — 所有算法迭代的速度上限由这里决定。

核心职责

  • A/B 测试平台 2.0（多变量、长期、交互式分析）
  • 特征平台 / Feature Store
  • 模型训练与在线服务基建
  • 实时数据管道与监控
  • 模型鲁棒性与质量保障流程

必备背景

  • 大规模 A/B testing 平台建设经验
  • ML infra / MLOps 经验
  • 对算法迭代痛点有第一性理解

典型来源

大型互联网公司实验平台团队 ML 平台 / MLOps 团队 独立 A/B testing 工具公司
P0.5 · 同步启动 · 架构

本地知识图谱与 Agent 运行时架构师

负责「现实世界结构化」+「AI agent 的工具与状态」 — 同时支撑信息平台与服务平台。

核心职责

  • POI / entity system / merchant graph
  • knowledge graph、entity resolution、geo 关系建模
  • AI agent runtime（工具调用、记忆、状态、编排）
  • LLM gateway / 模型路由 / 成本治理
  • 同时服务「信息侧搜索」与「服务侧 agent」

必备背景

  • LBS / map / geo system 经验
  • knowledge graph / large-scale entity system
  • LLM agent / 工具调用基建（加分）

典型来源

本地生活 POI / 商家中台 大型地图与知识图谱团队 LLM agent 平台公司
07

整体招聘优先级

六个核心岗位按优先级排列；P0 同步推进，缺一个都会卡战略。

  1. P0 · 立即招聘
    本地意图与搜推融合 Lead 算法
    决定信息平台是否能从「内容推荐」升级为「本地决策系统」。
  2. P0 · 立即招聘
    本地 AI Agent 与行动优化 Lead 算法
    决定能否从「information」走向「action」 — 服务平台的核心算法。
  3. P0 · 立即招聘
    广告算法与商业化科学 Lead 算法
    支撑商业化与跨应用矩阵变现 — 战略 4 的核心算法位置。
  4. P0 · 立即招聘
    应用矩阵与孵化平台架构师 架构
    没有共享架构，矩阵就只是孤岛 — 战略 1 + 2 的物理基础。
  5. P0 · 立即招聘
    实验与 ML 平台架构师 架构
    决定所有算法迭代的速度上限 — 战略 4 显式强调的关键差距。
  6. P0.5 · 同步启动
    本地知识图谱与 Agent 运行时架构师 架构
    同时供给信息平台与服务平台的「现实世界结构化」+「agent 工具」基建。
08

关键结论

人才决定战略能否落地。当前最大风险不在「再做一个更好的新闻推荐」，而在：

缺少把本地内容升级为本地信息与服务的
算法新主线（意图 / Agent / 商业化）
与 架构新底座（应用矩阵 / 实验平台 / 知识图谱 + Agent 运行时）
的负责人。
